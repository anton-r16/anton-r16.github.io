OVERVIEW

Student Portfolio is a sub-page created for the exhibition of student projects done during the media computer science studies at the University of Applied Sciences Mittelhessen.Another objective involved working with Pimcore as a content management system. Therefore, a key consideration for any design decision was its feasibility for implementation within this CMS.It was to be designed to help potential employers discover student capabilities and furthermore to help potential students who are interested in media computer science to see the results of student activities. For accessibility mobile as well as desktop versions had to be addressed in a responsive manner.

GOALS

In this project, it was a goal to acquire an understanding of established user interface- and user experience practices to adjust them to what we were going to gather during user research. We found a qualitative approach that encompasses a competitive analysis focussed on used user interface elements to be useful for the former where in addition literature on the same topic was brought in. For the latter, the gathered information and assumptions about the presumed target audience were to be synthesized into Proto-Personas.

Who is the website for?
Who is competing in the same market?
Where do people post their work?
What does the target audience need the most?
What could the target audience have problems with?
Where can we gather information about user interface elements?

CONCLUSION

CONSIDERATION

When going trough student material which was available to be used within this portfolio project we came across the option to implement a 3D-viewer for student made models during that process.

Which was ideal, since theoretically we would be able to combine this possibility with the already decided on gallery-element as part of our visual design. Gallery-element -> Interactive 3D-viewer.

USER EXPERIENCE & PAIN POINTS

One of the pain points of potential hiring managers in the games industry could be to find potential employees who produce visually compelling 3D-assets, which are also technically viable to be used within a games engine. These assets are to be made for use in real time. So a portfolio has to be able to showcase this when 3D-assets are included.

To address this, the interactive gallery element has to include information about the technical properties of the showcased 3D-asset. This could be a so-called wireframe rendering that showcases the bare-bones geometry of the created asset without texturing. That way somebody will be able to see if an asset uses to much geometry to compute within the game engine efficiently. To solve this, a toggle or button element should be included to switch between the textured view and a view of the asset's technical properties.

On this groundwork, a user flow could be developed which then could be visually translated into a corresponding sketch, as seen in the attached image above. Making sure every step of the user journey was met within the sketch the same could be turned into a digital wireframe in the next step.
